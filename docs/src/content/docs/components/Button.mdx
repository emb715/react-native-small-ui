A `TouchableOpacity`-based button with variant support for intent and size. Copy and adapt to your design system.

export const component = `
import { TouchableOpacity, Text } from 'react-native';
import { createComponent } from 'react-native-small-ui';

const ButtonBase = createComponent(TouchableOpacity, {
  base: {
    alignItems: 'center',
    justifyContent: 'center',
    borderRadius: 8,
  },
  variants: {
    intent: {
      primary: {
        _light: { backgroundColor: '#007AFF' },
        _dark:  { backgroundColor: '#0A84FF' },
      },
      danger: {
        _light: { backgroundColor: '#e00c2c' },
        _dark:  { backgroundColor: '#be0a25' },
      },
      ghost: {
        backgroundColor: 'transparent',
        _light: { borderWidth: 1, borderColor: '#007AFF' },
        _dark:  { borderWidth: 1, borderColor: '#0A84FF' },
      },
    },
    size: {
      sm: { paddingVertical: 6,  paddingHorizontal: 12 },
      md: { paddingVertical: 10, paddingHorizontal: 20 },
      lg: { paddingVertical: 14, paddingHorizontal: 28 },
    },
  },
  defaultVariants: {
    intent: 'primary',
    size: 'md',
  },
});
`;

## Usage

```tsx
// Defaults apply — primary intent, md size
<ButtonBase onPress={handleSubmit}>
  <Text style={{ color: '#fff' }}>Submit</Text>
</ButtonBase>

// Explicit variants
<ButtonBase intent="danger" size="lg" onPress={handleDelete}>
  <Text style={{ color: '#fff' }}>Delete</Text>
</ButtonBase>

<ButtonBase intent="ghost" size="sm" onPress={handleCancel}>
  <Text style={{ color: '#007AFF' }}>Cancel</Text>
</ButtonBase>
```

## Platform-specific shadow

```tsx
import { TouchableOpacity, Text } from 'react-native';
import { createComponent } from 'react-native-small-ui';

const Button = createComponent(TouchableOpacity, {
  paddingVertical: 10,
  paddingHorizontal: 20,
  borderRadius: 8,
  alignItems: 'center',
  _light: { backgroundColor: '#007AFF' },
  _dark:  { backgroundColor: '#0A84FF' },
  _ios:     { shadowColor: '#000', shadowOpacity: 0.2, shadowRadius: 4, shadowOffset: { width: 0, height: 2 } },
  _android: { elevation: 4 },
});

<Button onPress={onPress}>
  <Text style={{ color: '#fff' }}>Press me</Text>
</Button>
```

## With slots — `Button.Text`

`.withSlots()` attaches named sub-components as dot-notation properties. Slots share reactive context (colorMode, breakpoints) with the parent automatically — no prop drilling needed.

Slots do not automatically inherit the parent's variant state, so `Button.Text` defines its own `intent` variant mirroring `Button`'s. `defaultVariants: { intent: 'primary' }` means the common case needs no extra prop — only non-default intents require an explicit value.

export const componentSlots = `
import { TouchableOpacity, Text } from 'react-native';
import { createComponent } from 'react-native-small-ui';

const Button = createComponent(TouchableOpacity, {
  base: {
    alignItems: 'center',
    justifyContent: 'center',
    borderRadius: 8,
  },
  variants: {
    intent: {
      primary: {
        _light: { backgroundColor: '#007AFF' },
        _dark:  { backgroundColor: '#0A84FF' },
      },
      danger: {
        _light: { backgroundColor: '#e00c2c' },
        _dark:  { backgroundColor: '#be0a25' },
      },
      ghost: {
        backgroundColor: 'transparent',
        _light: { borderWidth: 1, borderColor: '#007AFF' },
        _dark:  { borderWidth: 1, borderColor: '#0A84FF' },
      },
    },
    size: {
      sm: { paddingVertical: 6,  paddingHorizontal: 12 },
      md: { paddingVertical: 10, paddingHorizontal: 20 },
      lg: { paddingVertical: 14, paddingHorizontal: 28 },
    },
  },
  defaultVariants: {
    intent: 'primary',
    size: 'md',
  },
}).withSlots({
  // Button.Text mirrors the parent intent variant so text color
  // stays in sync without raw color props at every call site.
  Text: createComponent(Text, {
    fontSize: 15,
    fontWeight: '600',
    variants: {
      intent: {
        primary: { _light: { color: '#fff' },     _dark: { color: '#fff' } },
        danger:  { _light: { color: '#fff' },     _dark: { color: '#fff' } },
        ghost:   { _light: { color: '#007AFF' },  _dark: { color: '#0A84FF' } },
      },
    },
    defaultVariants: { intent: 'primary' },
  }),
  Icon: createComponent(Text, {
    fontSize: 16,
    marginRight: 6,
    variants: {
      intent: {
        primary: { _light: { color: '#fff' },     _dark: { color: '#fff' } },
        danger:  { _light: { color: '#fff' },     _dark: { color: '#fff' } },
        ghost:   { _light: { color: '#007AFF' },  _dark: { color: '#0A84FF' } },
      },
    },
    defaultVariants: { intent: 'primary' },
  }),
});
`;

<Code title="Button.tsx" code={componentSlots} lang="tsx" />

```tsx
// primary is the default — Button.Text needs no intent prop
<Button intent="primary" size="md" onPress={onPress}>
  <Button.Icon>★</Button.Icon>
  <Button.Text>Confirm</Button.Text>
</Button>

// danger — pass intent explicitly on the slot
<Button intent="danger" size="sm" onPress={onDelete}>
  <Button.Text intent="danger">Delete</Button.Text>
</Button>

// ghost — text color switches automatically via the variant
<Button intent="ghost" onPress={onCancel}>
  <Button.Text intent="ghost">Cancel</Button.Text>
</Button>
```

The `intent` prop on `Button.Text` is type-safe and autocompleted — a wrong value is a TypeScript error. Slots share **colorMode and breakpoints** with the parent and re-render together when the color scheme changes.
